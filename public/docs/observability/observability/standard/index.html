    Observability Standard#

    本文档定义了 Metric / Log / Trace 在 Observability 系统中的标准 Labels / Attributes，用于统一收集、查询和关联数据。

    3. Metric / Trace / Log 公共 Attributes#

    Key 类型 说明 示例 适用类型
    trace_id string Trace ID，用于关联 Trace / Log / Metric 2e71abb92e031efc2a7a1c4280959f4b Trace / Log
    http.method string HTTP 方法 GET Metric / Trace / Log
    http.route string 路由模板 /users/:id Metric / Trace / Log
    http.status_code int HTTP 状态码 200 Metric / Trace / Log
    http.server.duration duration 请求耗时 120ms Metric / Trace / Log

    4. Trace / Log 特有 Attributes#

    Key 类型 说明 示例 适用类型
    span_id string Span ID，仅 Trace 使用 abc123def456 Trace
    parent_span_id string 父 Span ID，仅 Trace 使用 xyz789ghi012 Trace
    error.message string 错误信息（如有） database timeout Trace / Log
    client.ip string 客户端 IP 地址 10.0.0.1 Trace / Log
    user_agent.original string User-Agent curl/7.68.0 Trace / Log

    5. Metric 特有 Attributes#

    Key 类型 说明 示例
    metric.name string 指标名称 http_request_duration_seconds
    metric.unit string 单位 ms / requests
    metric.type string 类型 gauge / counter / histogram

    注意：HTTP 字段 (http.method / http.route / http.status_code / http.server.duration) 可作为 Metric Label 使用，方便拆分指标。

    6. Log 特有 Attributes#

    Key 类型 说明 示例
    log.level string 日志级别 INFO / WARN / ERROR
    log.logger string 日志记录器名称 gin / otelzap
    log.message string 日志文本 http request completed
    log.stacktrace string 可选，错误堆栈 at main.go:45

    7. Trace 特有 Attributes#

    Key 类型 说明 示例
    span.name string Span 名称 Tekton.CreatePipelineRun
    span.kind string Span 类型 server / client / producer / consumer
    span.status string Span 状态 ok / error

    8. 统一规范#

    1. 同一请求生命周期内的 Metric / Log / Trace 应共享以下 Attributes：

      • service.name
      • service.version
      • service.instance.id
      • deployment.environment
      • k8s.pod.name
      • trace_id（如果有 Trace）

    2. 命名规范：

      • Attribute 使用 小写 + 点号分隔 (service.name)
      • Kubernetes 相关字段加前缀 k8s.
      • HTTP 相关字段加前缀 http.
      • 日志相关字段加前缀 log.
      • Span 相关字段加前缀 span.

    3. 来源要求：

      • 所有 Attribute 应来源于 环境变量 或 资源信息，避免硬编码。
