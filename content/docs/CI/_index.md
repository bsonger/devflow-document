# 🚀 持续集成（CI）概览

CI 的目标是把「代码变成可信可交付的制品」，强调 **自动化、可追溯与安全可信**。

## 🎯 核心目标

- 让每次提交都可构建、可测试、可发布  
- 让制品可追溯（Git Commit / Image Digest）  
- 让安全前置（SAST / SBOM / 签名 / 漏洞扫描）  
- 让反馈及时（通知 / 指标 / Trace）  

## 🧱 核心能力

- 触发与编排：Pipeline / Task / Trigger  
- 构建与制品：镜像构建、仓库管理  
- 质量与安全：测试、扫描、签名  
- 可观测：日志、指标、Trace
